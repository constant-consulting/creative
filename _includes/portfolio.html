Software Delivery
Every project is a snowflake, and it needs its right toolset, right language and a right approach. We work with multiple languages and approach. Also, buiding alone has no business value, it should be delivered.
Inceptions and project planning
Have an idea that needs a roadmap? A short inception should give a high level idea, a crows view of what is to be done and a rough estimations and release planning. We can help you plan your software from your idea while keeping in mind that change is inevitable.
Team Culture
Software delivery is a team activity. It might need mentor-ship by example, technology ramp up and above all team culture. We can help build a team culture that focusses more on value based delivery rather than statistics based delivery.
Test automation
A repeatetive guarantee of what works gives a confidence to be able to deliver something and be sure that it works. Once the size of the project grows, so does the areas that a change can impact. This is where test automation keeps the team sane and productive.
Continuous Delivery
Ever spend money on a feature only to wait months before it hit the market? Delivery is often an overlooked aspect. This problem can be solved by bringing both aspects of work to the same team. Its all about building platform on demand, making sure it can be repeated when it is needed.
